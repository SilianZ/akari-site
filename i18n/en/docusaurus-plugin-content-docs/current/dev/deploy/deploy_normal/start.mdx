# Run Bot

AkariBot is mainly composed of platform robots. In order to enable developers to better test modules, we also provide a testing console. Next, we will gradually explain how to run the bot.

## Run Platform Bot

```mdx-code-block
<Tabs groupId="operating-systems">
<TabItem value="win" label="Windows" default>
```

We do not recommend double-clicking on `start.bat` to start the program.

It is recommended to open the terminal (cmd or Powershell) before running `start.bat`.

1.  In the directory where `start.bat` is located,  press `Shift` + `Right-click` to open the right-click menu.
2.  Open the terminal.
3.  Enter `.\start.bat` (Powershell) or `start.bat` (cmd) inside the terminal to start the bot.

```mdx-code-block
</TabItem>
<TabItem value="linux" label="Linux">
```

1.  In the terminal, set the execution permission for the 'start' script to 'chmod+x start'.
2.  Start script: `./start`.

## Run Test Console

The test console includes a basic runtime environment where you can use commands for basic robot interaction.

The test console only supports replies to text messages and images. Other message elements will be ignored or converted to text or images to show.

You may need to switch to a Poetry virtual environment using `poetry shell` to call the previously installed dependencies.

```mdx-code-block
<Tabs groupId="operating-systems">
<TabItem value="win" label="Windows" default>
```

1.  Press `Shift` + `Right-click` to open the right-click menu in the directory where `console.py` is located. 
2.  Open the terminal.
3.  Enter `python console.py` in the terminal to start the test console.

```mdx-code-block
</TabItem>
<TabItem value="linux" label="Linux">
```

1.  Open the terminal in the directory where 'console.py' is located.
2.  Enter `python console.py` in the terminal to start the test console.
